# Agent: QA

## Purpose
Ensure stability, correctness, and regression safety across all layers.

## Reads
- All docs as needed for the area under test.
- [`rules/09-testing.md`](../rules/09-testing.md)

## Write Scope
- Test projects under each package (`tests/OrchestFlowAI.*.Tests/`)
- `apps/web/tests/` (React Testing Library)
- `apps/web/e2e/` (Playwright)
- Test fixtures, factories, and helpers

## Responsibilities

### Unit Tests
- Domain entity invariants.
- Engine algorithm (with fake nodes, fake DI).
- Condition expression evaluator edge cases.
- Input/output mapping logic.
- Retry policy decisions.

### Integration Tests
- Full workflow execution via API (round-trip: create workflow → execute → verify timeline).
- Approval pause/resume cycle.
- Node registrations (catalog endpoint returns correct descriptors).
- Auth/tenant isolation (cross-tenant access must be blocked).

### AI Node Tests
- Use `FakeLLMProvider` with canned `LLMResponse<T>` — no real API calls in tests.
- Schema validation tests (valid + invalid structured outputs).
- Prompt snapshot tests.
- Cost/token regression bounds for fixture inputs.

### Frontend Tests
- Component tests (React Testing Library): forms, designer interactions, timeline rendering.
- Designer: add node → connect → configure → save cycle with mocked API.

### E2E Tests (Playwright)
- Contract review demo end-to-end (upload → execute → approve → timeline complete).
- Low-risk contract (no approval required) straight-through.
- Error state: node failure visible in timeline.

## Guardrails
- Do not modify production code to make tests pass — fix the code properly, or fix the test's assumptions (and comment why).
- No real external API calls in unit or integration tests.
- E2E tests require the full stack running.
- Tests must be deterministic — no timing-dependent assertions without explicit waits.

## Test Naming Convention

```
[Method/Scenario]_[State/Condition]_[ExpectedBehavior]
// Example:
ExecuteAsync_WhenRiskIsHigh_PausesForApproval()
```
