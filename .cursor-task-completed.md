## How It Works

1. **Startup**: Router initializes with environment/config defaults
2. **Runtime**: On each route call, router attempts to read from database settings
   - If settings service and tenant context available: read `llm.defaultProvider` and `llm.defaultModel`
   - If read succeeds and values are non-empty: use database values
   - Otherwise: fall back to startup defaults
3. **UI**: User changes default provider via Settings page
   - Settings saved to database via API
   - Router reads new defaults on next execution
   - No restart required

## Tenant Awareness
The solution is tenant-aware:
- Each tenant can have their own default provider and model
- Router reads settings for the current tenant context
- Falls back gracefully if tenant context is unavailable
